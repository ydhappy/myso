# 에고무기 상대 캐릭터 감지 기능 가이드

## 1. 개요

에고무기가 내 화면/주변에 보이는 상대 캐릭터를 감지하고, 공개 정보와 추정 위험도를 알려주는 기능입니다.

목표:

- 타겟 중인 캐릭터 분석
- 가장 가까운 주변 캐릭터 분석
- 주변 캐릭터 목록 출력
- 혈맹/카오/PK/중립 관계 표시
- 클래스/성향/PK수/거리/HP구간/무기종류 표시
- 위험도와 행동 조언 출력

## 2. 추가 파일

```text
ego/java/EgoOpponentScanController.java
```

실제 적용 위치:

```text
bitna/src/lineage/world/controller/EgoOpponentScanController.java
```


## 4. 사용 명령어

### 일반 채팅

에고 이름이 `에고`일 때:

```text
에고 상대
에고 적
에고 타겟분석
에고 주변캐릭
에고 플레이어
에고 위험도
```

에고 이름을 `카르마`로 만들었다면:

```text
카르마 상대
카르마 주변캐릭
카르마 타겟분석
```

### 점 명령어

```text
.에고상대
.에고주변
```

## 5. 분석 방식

### .에고상대 / 에고 상대

1순위:

```text
현재 내가 타겟으로 잡고 있는 PcInstance
```

2순위:

```text
내 주변에서 가장 가까운 PcInstance
```

대상이 없으면:

```text
[에고] 감지 범위 안에 상대 캐릭터가 없습니다.
```

### .에고주변 / 에고 주변캐릭

주변 플레이어 목록을 최대 5명까지 출력합니다.

```text
[에고] 주변 캐릭터 3명 감지
1) [카오] 홍길동 [기사] / 거리 3 / 위험도 위험
2) [중립] 테스트 [요정] / 거리 6 / 위험도 낮음
3) [혈맹] 내혈원 [군주] / 거리 2 / 위험도 낮음
```

## 6. 출력 정보

상대 상세 분석 예:

```text
========== 에고 상대 감지 ==========
대상: [카오] 홍길동 [기사]
호칭: 전장의왕
혈맹: 적혈 / 성향: Chaotic / PK: 7
거리: 3칸 / HP상태: 보통 / 무기종류: 양손검
위험도: 매우위험 / 판단: 정면 교전은 피하고 거리 확보를 권합니다.
```

## 7. 공개 정보 범위

표시하는 정보:

```text
캐릭터 이름
호칭
혈맹명
클래스
성향
PK 카운트
거리
HP 구간
상대 착용 무기 종류
위험도
에고 조언
```

표시하지 않는 정보:

```text
상대 계정
상대 IP
상대 인벤토리 전체
상대 정확한 HP 숫자
상대 정확한 스탯
상대 전체 장비명
상대 숨김 버프 전체
운영자 내부 정보
```

## 8. HP 표시 정책

PvP 밸런스 보호를 위해 상대 HP는 정확한 숫자가 아니라 구간으로 표시합니다.

```text
0~25%     위험
26~50%    낮음
51~75%    보통
76~100%   높음
```

## 9. 위험도 계산 기준

위험도는 다음 요소를 종합해서 계산합니다.

```text
상대 레벨이 나보다 높은가
거리가 가까운가
카오틱 상태인가
PK 카운트가 있는가
상대 무기 종류가 공격적인가
```

위험도 등급:

```text
낮음
주의
위험
매우위험
```

## 10. 관계 표시

```text
[혈맹] 같은 혈맹
[카오] 카오틱 성향
[PK] PK 카운트 있음
[중립] 특별 위험 표시 없음
```

## 11. 클래스 표시

기존 서버의 클래스 상수를 사용합니다.

```text

[요정]
[마법사]
[다크엘프]
```

## 14. 컴파일 오류 대응

### getClassType 오류

`PcInstance` 또는 상위 `Character`에 클래스 타입 getter가 없다면, 기존 `/누구` 코드에서 사용하는 메서드명을 확인하세요.
현재 저장소의 `C_ObjectWho.java`는 `use.getClassType()`를 사용합니다.

### getClanId/getClanName 오류

기존 `/누구` 코드에서 `use.getClanId()`, `use.getClanName()`를 사용하므로 현재 저장소 기준으로 존재합니다.

### getPkCount/getLawful 오류

기존 `/누구` 코드에서 `use.getPkCount()`, `use.getLawful()`를 사용하므로 현재 저장소 기준으로 존재합니다.

### getInsideList 오류

주변 객체 탐색에 사용합니다.
서버 버전에 따라 메서드명이 다르면 `object.java`에서 insideList getter 이름을 확인해 수정하세요.

## 15. 밸런스 주의

상대감지는 너무 강력하면 PvP 밸런스를 해칠 수 있습니다.
그래서 1차 구현은 다음 정보를 숨깁니다.

```text
정확한 HP 숫자
정확한 장비명
정확한 스탯
상대 인벤토리
상대 버프 전체
IP/계정 정보
```

운영자가 원하면 GM 전용 상세 감지 기능을 별도로 만들 수 있지만, 일반 유저 에고 기능에는 넣지 않는 것을 권장합니다.
