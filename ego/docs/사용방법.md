# 에고무기 사용방법

## 1. 적용 기준

에고무기는 기존 무기 구조를 변형하지 않고 성장/대화/보조전투 기능만 추가합니다.

`bitna/빛나` 폴더는 참고용입니다. 배포와 적용 기준은 `ego/` 폴더이며, Java 배포 코드는 `ego/java` 기준으로 관리합니다.

```text
ego 패키지 배포 기준
bitna/빛나는 참고용
item.type2 변경 없음
아이템 템플릿 변경 없음
인벤토리/바닥 이미지 변경 없음
DamageController 기본 공식 유지
SQL은 ego/sql/ego_schema.sql 1개만 사용
문서는 사용방법.md, 업데이트내용.md 2개만 유지
```

## 2. SQL 적용

신규/기존 서버 모두 아래 파일 1개만 적용합니다.

```sql
SOURCE ego/sql/ego_schema.sql;
```

DB 툴에서 `SOURCE`가 동작하지 않으면 `ego/sql/ego_schema.sql` 내용을 전체 복사해서 실행합니다.

적용 후 서버에서 실행합니다.

```text
.에고리로드
```

## 3. Java 연결

`ego/java` 배포 코드를 서버 소스 구조에 맞게 복사한 뒤, 기존 서버 코드에는 가능하면 `EgoCore`만 연결합니다.

### 서버 시작

```java
EgoCore.init(con);
```

### 리로드

```java
EgoCore.reload(con);
```

### 명령어 연결

```java
if (EgoCore.command(o, key, st)) {
    return true;
}
```

`void` 구조라면:

```java
if (EgoCore.command(o, key, st)) {
    return;
}
```

### 일반 채팅 연결

주변 채팅 방송 전에 호출합니다.

```java
if (EgoCore.chat(o, msg)) {
    return;
}
```

### 주기 처리

캐릭터 상태 루프나 AI 루프에서 호출합니다.

```java
EgoCore.tick(pc);
```

### 공격 보정

`DamageController.getDamage(...)` 최종 반환 직전에 연결합니다.

```java
if (cha instanceof PcInstance && weapon != null && dmg > 0) {
    dmg = EgoCore.attack((Character) cha, target, weapon, (int) Math.round(dmg));
}
```

### 피격 보정

HP 감소 직전에 연결합니다.

```java
if (o instanceof Character) {
    dmg = EgoCore.defense((Character) o, cha, dmg);
    if (dmg <= 0)
        return;
}
```

## 4. 게임 명령어

```text
.에고도움
.에고생성 이름
.에고삭제 확인
.에고정보
.에고이름 이름
.에고말투
.에고능력
.에고상대
.에고주변
.에고리로드
```

`.에고생성 이름`은 무료입니다. 생성 시 능력과 대화 성향은 랜덤으로 정해지고 이후 유지됩니다.

`.에고삭제 확인`은 무료입니다. 착용 중인 해당 에고무기 정보만 삭제합니다. 전체 DB 삭제/전체 테이블 삭제/전체 초기화 기능은 제공하지 않습니다.

`.에고능력`, `.에고말투`는 현재 상태 확인과 안내만 합니다. 능력/대화 성향 변경은 명령어가 아니라 `에고 변경구슬` 아이템 사용으로만 처리합니다.

## 5. 에고 변경구슬

`에고 변경구슬`은 착용 중인 에고무기의 능력과 대화 성향을 랜덤으로 다시 선택하는 소모 아이템입니다.

```text
아이템명: 에고 변경구슬
기본 item_code: 900001
Java 클래스: lineage.world.object.item.EgoChangeOrb
사용 조건: 에고무기 착용 상태
소모: 사용 시 1개
효과: 능력 랜덤 재선택 + 대화 성향 랜덤 재선택
```

SQL은 `ego_item_template`에 기준값을 저장합니다. 서버마다 `item` 테이블 필수 컬럼이 다르기 때문에 `ego_schema.sql`은 실제 `item` 테이블에 직접 INSERT하지 않습니다. SQL 실행 결과에 출력되는 `ego_item_template` 값을 참고해 서버의 `item` 테이블 구조에 맞게 직접 등록합니다.

아이템 클래스 연결은 서버의 `ItemDatabase` 생성 분기에서 `에고 변경구슬` 또는 item_code `900001`을 `lineage.world.object.item.EgoChangeOrb`로 생성하도록 연결해야 합니다.

## 6. 인벤 표시

인벤토리 아이템명에는 `[에고]`와 종류 문구를 붙이지 않습니다.

```text
싸울아비 장검 [Lv.5 카르마]
```

기존 표시가 남아 있는 경우 `.에고리로드` 또는 인벤 갱신 시 구표식 `[에고]`, 구버전 `[Lv.]` suffix를 제거하고 새 형식으로 다시 표시합니다.

## 8. 성장 규칙

```text
생성 레벨: 0
최대 레벨: 10
Lv.0: 전투능력 없음
Lv.1: 기본 능력 발동 시작
Lv.5: 피격 반격 시작
Lv.6: 자동반격 시작
Lv.10: 스턴 연동
```

기본 경험치:

```text
공격 중 3초마다 +1
몬스터 처치 +5
보스 처치 추가 +50
```

수치는 `ego_config`, `ego_level`, `ego_skill_base`, `ego_weapon_rule`에서 조정합니다. 변경 후 `.에고리로드`를 실행합니다.
