# 에고무기 업데이트내용

## 2026-04-28 단순 재정립

### 최종 구조

```text
SQL 파일: ego/sql/ego_schema.sql 1개

```

### 핵심 정책

```text
점명령 생성/삭제/변경 제거
에고 생성은 에고 구슬 아이템 사용으로만 처리
에고 구슬은 착용 무기에 에고가 없을 때만 최초 생성
이미 에고무기면 능력/대화/레벨/경험치 변경 없음
에고무기가 다른 캐릭터에게 이동되면 현재 캐릭터를 주인으로 자동 재인식
무기 타입별 허용/차단 규칙 제거
무기 슬롯 아이템이면 에고 대상, 낚싯대만 제외
```

### 능력명 정책

```text
Java 전투 컨트롤러 런타임 능력명과 DB 저장 능력명을 1:1로 유지
단순명 입력은 EgoWeaponDatabase에서 런타임 능력명으로 정규화
```

### 보완 내용

```text
EgoCore.java: 점명령 진입점 제거, init/reload/chat/tick/attack/defense/useOrb만 유지, 구슬 처리/재인식 내부 병합

EgoWeaponTypeUtil.java: 타입별 규칙 제거, 무기 슬롯 + 낚싯대 제외만 판정, 기본 능력명은 런타임명으로 반환
EgoSchema.java: 현재 SQL 기준으로 스키마 진단표 정리, ego_weapon_rule 제거, ego_bond/ego_item_template 포함
EgoLevel.java: ego_level 테이블만 읽도록 정리, ego_level_exp/ego_level_bonus fallback 제거
EgoView.java: [에고]와 종류 문구 제거, [Lv.값 에고이름]만 표시
EgoMessageUtil.java: 줄바꿈 후 색상 유지, HTML fallback 안정화
ego_schema.sql: ego_weapon_rule/ego_level_exp/ego_level_bonus 제거, ego_bond fallback 유지, 런타임 능력명 기준 데이터 유지
```

## 운영 주의

전체 삭제/전체 초기화 SQL은 제공하지 않습니다.

에고 삭제 명령도 제공하지 않습니다. 운영자가 수동으로 정리해야 할 경우에는 반드시 대상 item_id를 확인한 뒤 DB에서 개별 정리해야 합니다.

에고 구슬 실제 아이템 등록은 서버별 item 테이블 구조에 맞춰 `ego_item_template` 값을 참고해 수동 등록합니다.
